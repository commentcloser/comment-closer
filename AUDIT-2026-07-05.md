# Comment Closer — Takeover Audit (2026-07-05)

Multi-agent audit of the inherited codebase at the moment of takeover (commit `d1e41be`, single squashed "Initial commit"). Every HIGH finding below was independently re-verified against the code by a second agent; one claimed HIGH was refuted (dead code) and is listed under Dead Code instead.

**Product:** Users connect Facebook Pages / Instagram / TikTok / TikTok Ads; comments arrive via webhooks (FB/IG/TikTok organic) and a polling cron (TikTok Ads); OpenAI (`gpt-5`) classifies sentiment; negative comments are auto-hidden/deleted; positive/neutral get AI replies (brand tone, cooldowns, block/allowlists, optional manual review, optional delay). Dashboard: unified inbox, per-page live browser, per-page AI settings, status page, admin area. i18n en/el (complete, 1002 keys each). Prod domain (from repo evidence): `www.commentcloser.com`.

**Overall:** The core pipeline is genuinely built and mostly works. But there are 4 critical security holes, the paid plans are pure fiction (no Stripe code at all), several features silently fail in production, and both cron jobs are effectively unscheduled.

---

## 1. CRITICAL security holes (all CONFIRMED, production-exploitable)

1. **Account takeover chain.** `app/api/auth/set-linking-user/route.ts` has NO auth — anyone can set the `linking_user_id` cookie to any userId. The Facebook `signIn` callback in `lib/auth.ts:141-191` then links a fresh FB account to that userId and **rewrites the session identity to the victim** → attacker gets a session as the victim. Victim userIds are harvestable via #3. Fix: require session and `session.user.id === userId` in set-linking-user.
2. **Unauthenticated account-link theft.** `app/api/facebook/link-account/route.ts:24` takes `targetUserId` from the request body with priority over the session, no auth when present. Attacker can steal a just-completed FB OAuth connection (incl. access token) during the 10-minute window.
3. **Cross-tenant data leak.** `app/api/debug/user-accounts/route.ts` requires only any logged-in session (self-signup is open) and returns **every user's email, userId, and FB providerAccountId**. All sibling debug routes correctly use `requireAdmin` — this one doesn't.
4. **Fully public data endpoint.** `app/api/webhooks/debug/route.ts` — no auth at all. Returns all tenants' connected IG pages + recent ad-comment text/authors to any anonymous visitor. NOTE: it lives under `api/webhooks/`, not `api/debug/`, so folder-level protection would miss it.
5. **Login brute-force is unthrottled.** `lib/auth.ts authorize()` imports `isRateLimited` but never calls it; only the client-side pre-check exists, bypassed by POSTing directly to `/api/auth/callback/credentials`. Also: rate limiting everywhere is an in-memory Map — meaningless on Vercel serverless (needs Upstash/DB).
6. **Unmetered AI spend by any user.** `app/api/comments/[id]/suggest-reply` has no ownership check — any account can generate replies for ANY comment id (cross-tenant leak + unlimited gpt-5/web_search burn).
7. Lesser: `test-facebook-config` + `check-facebook-config` unauthenticated (leak secret lengths + NEXTAUTH_URL); `debug-token` returns raw page tokens to the browser; all Meta/TikTok tokens stored plaintext and passed as URL query params; hardcoded dev ad account `act_269316045245432` ("Elyon Mon IKE") prioritized for every user in `app/api/facebook/pages/route.ts:630-651,853-872`.

## 2. Billing is 100% vaporware (CONFIRMED)

- Landing page sells Free $0 / Pro $49 / Business $80 with per-plan AI-reply quotas (50/1,000/2,000). **Every CTA links to free /register.** Zero Stripe imports anywhere (3 Stripe deps unused). No subscription/plan/usage fields in Prisma. No quota enforcement anywhere in the pipeline — every signup gets unlimited gpt-5 usage.
- **Terms and Privacy pages make false claims** ("billed monthly via Stripe", "We store your Stripe customer ID") — compliance + Meta/TikTok app-review risk.
- The intended design is fully recoverable from ~120 lines of orphaned i18n strings (`billingBanner`, `userBilling`, `pricing`, admin `billing` namespaces): card-on-file free tier, auto-upgrade to Pro at 50 comments, customer portal, dunning banners, admin MRR/ARR dashboard.

## 3. Silently broken features (CONFIRMED)

1. **Delayed replies never post.** The only scheduler is `.github/workflows/post-scheduled-replies.yml`, which curls prod every 5 min **without the required `Authorization: Bearer CRON_SECRET` header** → every run 401s, workflow stays green. `vercel.json` is `{}` (no Vercel crons).
2. **TikTok Ads ingestion depends on an unknown cron-job.org account** (referenced only in a code comment at `fetch-tiktok-ads-comments/route.ts:84`). If it lapses, ingestion silently stops.
3. **Organic TikTok delayed replies are impossible:** `post-scheduled-replies` has no `tiktok` branch — posts TikTok comment IDs to graph.facebook.com, always fails, reply silently dropped.
4. **TikTok Ads manual-review approvals always fail:** `approve-reply` has no `tiktok_ads` branch (same Facebook-branch fallthrough).
5. **Email likely misconfigured:** `lib/email.ts` falls back to `onboarding@resend.dev` when `EMAIL_FROM` unset (only delivers to the Resend account owner), and with resend v6 the `{data, error}` result is never checked → send failures return success silently. Missing RESEND_API_KEY = silent success, no logging (dev flows untestable).
6. **Decision-engine bypass on Meta webhook redelivery:** skipped comments stay `status='pending'`; a redelivered/edited-comment event calls `generateAndPostAutoReply` without re-running cooldown/blocklist/first-comment rules → rejected comments get replies anyway.
7. **Web-search failures post a hardcoded GREEK fallback** publicly to any-language commenters on web-enabled pages; web_search domain restriction is prompt-text only.
8. Onboarding FB-connect uses `setTimeout` chains — intermittent "no pages" race on slow networks.

## 4. Cost bomb

- `gpt-5` (flagship, 13 occurrences) for EVERYTHING including one-word sentiment classification; no max_tokens, no reasoning-effort caps on the main paths, `tokensUsed` never persisted, no metering/quota anywhere. A viral post = thousands of unmetered flagship calls. Quick win: sentiment → `gpt-5-mini`/nano (~1/25 cost), add output caps + usage metering.
- Sentiment errors return null → comments stuck `pending` forever, no retry/backfill. `attemptCount` incremented but never read (no circuit breaker). No OpenAI 429 retry.

## 5. Database / migrations (dangerous for us as new devs)

- **~40 columns + entire TikTokAccountStats table exist in schema.prisma but in NO migration** (prod maintained via `prisma db push` + hand-run SQL; loose hotfix .sql files committed). `prisma migrate deploy` on a fresh DB = broken schema.
- **NEVER run `npm run db:migrate` (= `prisma migrate dev`) against the prod DATABASE_URL** — drift detection will prompt to RESET (drop) the database. Remediate: `prisma migrate diff` → catch-up baseline → `migrate resolve --applied`.
- Local dev: create `.env` with `DATABASE_URL=postgresql://...?connection_limit=5`, then `npx prisma db push` (NOT migrate). `prisma.config.ts` exists → CLI doesn't auto-load .env except via its dotenv import.
- `dev.db` (root, 53KB) + `prisma/dev.db` (0B) committed to git, NOT gitignored. Verified schema-only today (no real emails/tokens), but any local login test would silently commit real credentials next time. `git rm --cached` + add `*.db` to .gitignore.

- No middleware.ts (dashboard protection is client-side only; API routes each self-guard — mostly correctly via `requireAdmin`/`requireCommentOwner`).
- No CI at all (only the broken cron workflow). No .env.example. No README.
- Landing page: placeholder "Company 1..5" logos, fabricated metrics/testimonials, dead "Watch Demo" button, footer links all `#`, footer brand says "AI Comment Replyer".
- Dead code: `contexts/AuthContext.tsx`, `lib/authMock.ts`, ~900-line `fetchAdsComments` in `app/api/facebook/comments/route.ts` (never called — the claimed HIGH race on this path was REFUTED as dead code), 5 orphaned locale sections, empty `logReplyDecision`, unused `hasPageAlreadyReplied`.
- Prior audit exists: `scalability-audit-report.html` (March 2026) — top risks (no DB pooling enforcement, synchronous webhook processing/no queue, admin OOM, no OpenAI retry) mostly STILL unfixed; only webhook HMAC + cron batch size were addressed.
- Positives worth keeping: webhook HMAC verification on all 3 platforms (timing-safe, fail-closed in prod), `requireAdmin` re-reads role from DB, strong token generation (crypto.randomBytes(32), bcrypt cost 12), never-reply-to-negative enforced at 3 layers, idempotent moderation via CommentActionLog, TikTok integration closely matches bundled offline API docs.
