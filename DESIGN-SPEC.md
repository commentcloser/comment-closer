# DESIGN-SPEC-V2 — "ULTRAVIOLET ENGINE"
## Comment Closer redesign V2 — authoritative visual system

Repo: `C:/Users/Elyon/Desktop/comment-closer`
Companion doc: `COPY-SPEC.md` (all copy + i18n keys). This doc is layout/style/motion; COPY-SPEC is words. Implementation agents make ZERO design decisions beyond these two files.

**Philosophy (read once, then obey):** This is a control room for money, not a brochure. Dark-first near-black indigo void. Exactly three things are allowed to glow: the problem (hot red `--u-danger`), the machine (electric violet `--u-accent`), the win (neon mint `--u-success`). One idea per viewport at billboard scale, 900-weight type, mono numerals for every number. Loud through scale/contrast/motion — never clutter. Premium through restraint everywhere else.

**Non-negotiable laws:**
1. **GLOW SCARCITY LAW: exactly ONE glowing object per viewport.** Hero → the TakeoverDemo card. Problem → the red money counter slab. Flip → the toggle switch. How → the active/step-4 card. Pricing → the Free (featured) card. Final CTA → the white button. Nothing else gets `shadow-pop`/`shadow-hard` in that viewport.
2. **One emotional color per section**: hero = violet, problem = red, flip = red→mint hand-off, how = violet, results = mint, pricing = violet, final CTA = inverted gradient.
3. **Every number on marketing pages is `font-mono`** (JetBrains Mono, tabular).
4. **No section may contain two competing headlines.** H1/H2 owns its viewport.
5. **Greek text is never `text-transform: uppercase`d in CSS.** Uppercase Greek strings are authored uppercase in `el.json`. Headline `line-height` is never below **1.05** (Greek diacritics Ά/ΐ/ϋ clip below that).
